# DCASE 2026 Task 1

Code for "Broad Sound Taxonomy Classification with Textual Metadata and Crowdsourced Annotations", currently under review for the DCASE
Workshop 2026.

## Summary

This work studies Broad Sound Taxonomy classification for heterogeneous
Freesound recordings using both audio and textual metadata. The approach
uses large-language-model-derived BST class probabilities from clip metadata
and fuses them with audio-text embeddings from CLAP-style models. It also
investigates whether the larger crowdsourced BSD35k-CS set can improve
training through soft pseudo-labels generated by models trained on the
cleaner BSD10k data. Experiments compare M2D- and CLAP-based classifiers and
show that LLM-derived metadata predictions are complementary to pretrained
text encoders, while simple pseudo-labeling is more useful than noisy labels
but does not surpass training on clean data alone.

## Setup

This repository requires Python 3.10 or newer.

```bash
conda create -n dcase2026-task1 python=3.10
conda activate dcase2026-task1
python -m pip install -e '.[dev]'
```

Log in to Weights & Biases if you want online experiment tracking:

```bash
wandb login
```

Some embedding backbones use external checkpoints. By default, the training
script looks for checkpoints in `checkpoints/` at the repository root. You can
override this with `--checkpoint-dir`.

## Data

Download the datasets from Zenodo:

- BSD10k: https://doi.org/10.5281/zenodo.17233904
- BSD35k-CS: https://doi.org/10.5281/zenodo.19187100

The processed metadata used by the metadata-aware experiments is available
here:

- https://cloud.cp.jku.at/index.php/s/ck7ksQJyopJbaMc

By default, the code expects the datasets at:

- `data/BSD10k`
- `data/BSD35k-CS`
- `data/BSD2k` for optional BSD2k evaluation/submission workflows

Each dataset root should contain the official `audio/` and `metadata/`
directories. You can also pass explicit paths:

```bash
python -m dcase2026_task1.experiments.training \
  --bsd10k-root /path/to/BSD10k \
  --bsd35k-root /path/to/BSD35k-CS
```

Place the downloaded `processed_metadata/` directory under `data/`.
If you use a different location, update the metadata paths in
`src/dcase2026_task1/data/datasets.py`.

## Run an Experiment

Run a single training experiment with the LAION-CLAP backbone:

```bash
python -m dcase2026_task1.experiments.training \
  --wandb-project=dcase2026-task1 \
  --wandb-mode=online \
  --embedding-model=lclap \
  --batch-size=18 \
  --learning-rate=4e-6 \
  --weight-decay=0.01 \
  --warmup-epochs=1 \
  --lr-decay-start-epoch=1 \
  --min-learning-rate=0 \
  --max-epochs=30
```

Run the LAION-CLAP model with LLM-prior embedding fusion:

```bash
python -m dcase2026_task1.experiments.training \
  --wandb-project=dcase2026-task1 \
  --wandb-mode=online \
  --embedding-model=lclap \
  --batch-size=18 \
  --use-llm-prior-embedding-fusion \
  --save-checkpoints \
  --learning-rate=4e-6 \
  --weight-decay=0.01 \
  --warmup-epochs=1 \
  --lr-decay-start-epoch=1 \
  --max-epochs=30
```

Set `--wandb-mode=offline` or `--wandb-mode=disabled` if you do not want
online logging. Training outputs are written to `output/` at the repository root
by default.

## Run a Sweep

The workshop sweep configuration for the stage-1 LCLAP + LLM experiment is:

```bash
scripts/workshop_sweeps/lclap_llm_stage1_bsd10k.yaml
```

Create the sweep:

```bash
wandb sweep scripts/workshop_sweeps/lclap_llm_stage1_bsd10k.yaml
```

Start an agent with the sweep ID printed by W&B:

```bash
wandb agent dcase2026-task1/<SWEEP_ID>
```
